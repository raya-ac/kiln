# Getting help with Kiln

Before opening an issue, it's usually faster to:

1. **Check the [latest release notes](https://github.com/raya-ac/kiln/releases)** — something you hit may already be fixed.
2. **Search [existing issues](https://github.com/raya-ac/kiln/issues?q=is%3Aissue)**, open and closed. A lot of questions have been answered once already.
3. **Ask in [Discussions](https://github.com/raya-ac/kiln/discussions)** — for usage questions, design chat, or "is this a bug or am I holding it wrong?"

## When to file an issue

File a [bug report](https://github.com/raya-ac/kiln/issues/new?template=bug_report.yml) when:

- Something crashes, hangs, or produces wrong output.
- You can reproduce it in a known Kiln version.

File a [feature request](https://github.com/raya-ac/kiln/issues/new?template=feature_request.yml) when:

- Something should exist that doesn't, and you've thought about the shape a little. "Kiln should be better" isn't enough.

## What not to file publicly

Security issues go privately to **security@raya.li** — see [SECURITY.md](SECURITY.md).

## Response time

This is one person working on it in their spare time. I usually get to issues within a few days, but "usually" isn't always. If something's blocking you and urgent, say so in the title.
